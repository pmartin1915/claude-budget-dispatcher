# Deep Research Phase 2: Technical Planning & Audit of Metacognition Architecture

## Objective
You are tasked with executing a rigorous technical audit and developing a concrete implementation plan for the "Metacognition and Autonomous Self-Repair Layer" of a decentralized Node.js fleet running on Windows. You must base your work entirely on the provided Architecture Decision Record ("Autonomous Dispatcher Self-Repair Design.txt").

## Context
The Phase 1 architecture mandates the following specific technologies and frameworks:
1. **Out-of-Band Watcher:** `Servy` (a Windows service manager) utilizing synchronous Pre-Launch hooks to intercept fatal Node.js crashes (Application Error 1000).
2. **Metacognitive Drift Engine:** A local Node.js process using `onnxruntime-node` (`all-MiniLM-L6-v2`) to convert JSONL logs into semantic vectors, calculating Exponential Moving Average (EMA) and Cosine Distance to detect logic looping (the "coherence trap").
3. **Autonomous Self-Repair Cascade:** A PowerShell script (`repair_orchestrator.ps1`) that orchestrates LLM API calls using Data Transformation Graphs (DTG) principles, delegating validation to a Node.js AST parser (`validate_patch.mjs`) inside an `isolated-vm` sandbox.
4. **Safety Boundaries:** Algorithmic Circuit Breakers (Closed/Open/Half-Open) stored in `circuit_state.json`, a regex Data Boundary Breaker to redact API keys from stderr, and a Global Dead Man's Switch via a GitHub Actions cron job.

## Your Mission
You must translate this high-level architecture into a highly detailed, line-level implementation plan, while actively auditing the design for technical impossibilities or security flaws.

### Deliverable 1: The Technical Implementation Blueprint
Provide step-by-step code structures, necessary file layouts, and configuration snippets for the following components:
- **`repair_orchestrator.ps1`**: Draft the PowerShell script that reads `dispatcher_stderr.log`, invokes the LLM API, and manages the circuit breaker state.
- **`validate_patch.mjs`**: Draft the Node.js utility utilizing Acorn/Babel to apply AST patches generated by the LLM, and the logic to spin up the `isolated-vm` for sandboxed testing.
- **Servy Configuration**: Provide the exact `Install-ServyService` hashtable and setup commands required for deployment on the Windows hosts.
- **Metacognitive Drift Logic**: Outline the exact array mathematics and `onnxruntime-node` initialization required for the EMA and Cosine Similarity calculation.
- **Global Watchdog**: Provide the exact `.github/workflows/global-watchdog.yml` structure.

### Deliverable 2: The Red Team Audit
You must critically audit the Phase 1 architecture. Specifically analyze:
1. **The Circuit Breaker Loophole:** Is the "Half-Open" state (testing a single repair after 2 hours) susceptible to a catastrophic infinite loop if the LLM continuously generates a patch that crashes exactly after the test passes? How do we prevent this?
2. **Servy Limitations:** Are there hidden limitations in Servy's Pre-Launch timeout bounds that would cause the LLM API call to silently fail or hang the Windows Service Control Manager (SCM)?
3. **Embedding Model Overhead:** Will `onnxruntime-node` running `all-MiniLM-L6-v2` exhaust the CPU or memory resources on a standard free-tier VPS alongside the primary Node.js dispatcher process?

Provide robust, technically sound mitigation strategies for any vulnerabilities you identify in your audit.
